## Why

The monorepo's dependencies have accumulated drift since the last major maintenance pass: several core libraries (MUI, MSAL, TypeScript, ESLint, Prisma, Fastify, TanStack, Vite, etc.) have minor, patch, and major versions available. Running on outdated dependencies increases security exposure (unpatched CVEs), blocks adoption of newer ecosystem features, and makes future upgrades harder because the gap keeps widening. We want to close the drift in a controlled, reversible way — one library at a time, verified end-to-end after each bump — so any regression is isolated to a single commit and trivially revertible.

## What Changes

- Upgrade all dependencies across the monorepo (`apps/api`, `apps/web`, `packages/*`, and the shared catalog in `pnpm-workspace.yaml`) to their latest compatible versions, grouped into three risk-tiered pull requests:
  - **PR 1 (low-risk)**: all patch + minor upgrades within the current major (Prisma 7.x, Fastify 5.x, TanStack Query/Router, Vitest, Tailwind, React 19.2.6, etc.).
  - **PR 2 (medium-risk)**: scoped majors (`@fastify/multipart` 10, `@vitejs/plugin-react` 6, `@testcontainers/*` 12, `jwks-rsa` 4, `ky` 2, `cpy-cli` 7, `globals` 17).
  - **PR 3 (high-risk)**: TypeScript 6, ESLint 10, MSAL 5, MUI core 9, MUI X 9 — applied in 5 internal phases, each with codemods and smoke tests.
- **Establish a per-library verification protocol**: after every single library version bump, run `pnpm install` and `pnpm build` (and `pnpm type-check && pnpm lint && pnpm test` per project rules) and, only if everything passes, create a dedicated commit for that library. Each commit is atomic and revertible.
- **BREAKING**: PR 3 introduces TypeScript 6 (stricter type inference may reveal latent type errors), ESLint 10 (config-shape changes), MUI core 9 (theme augmentation and `Grid` API changes), and MSAL 5 (auth client configuration changes). These are behind a feature flag of "merge order" — they ship last and only after PR 1 and PR 2 are merged and stable.

## Impact

- **Affected files**:
  - `pnpm-workspace.yaml` (catalog versions).
  - All `package.json` files: root, `apps/api/`, `apps/web/`, `packages/constants/`, `packages/database/`, `packages/eslint-config/`, `packages/types/`, `packages/typescript-config/`, `packages/utils/`.
  - `pnpm-lock.yaml` (regenerated by `pnpm install`).
  - PR 3 only:
    - `apps/web/src/**` (MUI codemods, MUI X codemods, MSAL config refactor).
    - `apps/web/src/api/http/client.ts` (`ky` v2 + MSAL token hook in PR 2 / PR 3).
    - `packages/eslint-config/{base,api,web}.ts` (ESLint 10 config adjustments).
    - `packages/typescript-config/*.json` (potential TS 6 option adjustments).
    - `apps/web/src/theme/palette.ts`, `apps/web/src/undp-huella-latam.theme.d.ts` (MUI 9 theme augmentation).
- **APIs**: no public API contract changes. The Fastify routes, Zod schemas, and Prisma models keep the same shape; only their underlying libraries change.
- **Dependencies**: every direct dependency in scope (see PR tables in `tasks.md`).
- **Systems**:
  - CI pipeline must keep passing at every commit (lint zero-warnings, type-check, tests, build).
  - Local dev workflow (`pnpm dev:api`, `pnpm dev:web`) must keep working after each PR.
  - Auth flow (MSAL + JWKS validation) is touched in PR 2 (`jwks-rsa`) and PR 3 (`msal-browser`/`msal-react`) — requires manual smoke test before merge.
  - Test infrastructure (`@testcontainers/*` 12) — must keep integration tests green.
